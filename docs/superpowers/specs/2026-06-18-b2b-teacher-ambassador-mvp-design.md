# B2B «Учитель-амбассадор» — MVP для валидации (дизайн-спека)

Дата: 2026-06-18 · Приложение: «Путь культивации» (Alchemist), Expo/react-native-web + Supabase.

## 1. Контекст и проблема
Оценка реалистичности монетизации показала: для дохода уровня зарплаты узкое место — **дистрибуция**, а не
приложение или платежи. Контент-light путь — **заимствовать чужую аудиторию через учителей**. Выбранная
модель: **ученики платят (подписка), учителя продвигают** приложение своим ученикам ради двух стимулов —
(1) **дашборд** (видят выполнение → лучше ведут, ученики дисциплинированнее) и (2) **rev-share**
(регулярный % с подписок своих учеников). Эта спека описывает **минимальный MVP**, который валидирует
модель на **1–3 реальных учителях** до полноценной разработки.

## 2. Цель и критерии готовности
- Учитель включает «режим учителя», получает реф-ссылку, опционально задаёт рекомендованную программу
  практик, видит приватный дашборд выполнения + статус оплаты своих учеников.
- Ученик заходит по ссылке учителя (атрибутируется), пользуется трекером, **оформляет ежемесячную
  подписку** (Telegram Stars).
- Rev-share учителя вычислим (выплата вручную).
- **Гейт решения:** ≥1–2 учителя реально продвигают (шарят ссылку, открывают дашборд) и их ученики
  подписываются → строим полную версию. Иначе — пивот/стоп.

## 3. Вне объёма (MVP не делает)
Авто-выплаты rev-share (только ручной расчёт); школы/мульти-учительские иерархии; роли сверх
«учитель/ученик»; глубокая аналитика; доставка контента (видео/аудио) в приложение; веб-платёжный
провайдер (только Telegram Stars); защита от накруток сверх базовой атрибуции; расширенный набор Premium
(на MVP — узкий набор, см. §6).

## 4. Модель (резюме)
- **Платит ученик** — ежемесячная подписка Premium через Telegram Stars.
- **Учитель продвигает** ради: (a) **пользы** — дашборд выполнения учеников; (b) **денег** — регулярный
  rev-share (% с подписок его учеников).
- **Атрибуция** — по личному реф-коду учителя (deep-link в Mini App). Привязка ученика к учителю —
  one-touch, на первом входе, неизменна.

## 5. Архитектура
- Работает как **Telegram Mini App** — то же веб-приложение, открытое внутри Telegram; оплата Stars.
  Standalone-сайт остаётся бесплатным (без платного потока) — это нормально для MVP.
- **Supabase:** Auth (через Edge Function `telegram-auth`), Postgres (текущая `user_data` + новые таблицы),
  Edge Functions (`telegram-auth`, `telegram-webhook`).
- **Платёжный фундамент (по дизайну Фазы 0) строится ЗДЕСЬ** — как подписка: таблица `entitlements`,
  Stars-оплата, Edge Function-webhook, хук `usePremium()`. На момент спеки он ещё **не реализован** —
  это часть данного MVP, а не готовый кирпич.

## 6. Компоненты и потоки

### 6.1 Учитель
- Включает «режим учителя» → создаётся строка `teachers` с `referral_code`.
- (Опц.) задаёт **рекомендованную программу**: выбирает/создаёт практики (переиспользуем `EditorSheet`) →
  сохраняется как `teachers.program` (jsonb).
- **Дашборд:** список привязанных учеников × {сегодня выполнено?, страйк, % за неделю, платит/нет}, плюс
  число платящих и **примерный заработок** (rev-share).
- Делится deep-link: `t.me/<bot>?startapp=<referral_code>`.

### 6.2 Ученик
- Открывает deep-link учителя → `telegram-auth` (валидация `initData`) → сессия Supabase → при первом входе
  записывается `user_data.referred_by_teacher = <teacher_id>` (один раз, неизменно).
- Практики из `teachers.program` добавляются в его «Сегодня» (помечены источником-группой).
- Пользуется трекером бесплатно; упирается в Premium-гейт → **Paywall** → оформляет подписку (Stars).
- Ежедневно клиент пишет **сводку выполнения** (`done/total/streak`) в `group_adherence`.

### 6.3 Premium для ученика (MVP-набор)
Подписка разблокирует Premium. На MVP набор узкий, но с регулярной ценностью:
- **Безлимит кастомных практик** (бесплатно ≤10; гейт в `savePractice`/`EditorSheet`).
- **Ежедневные напоминания о практике через бота** (Telegram-сообщения — дёшево в Mini App контексте,
  даёт регулярную ценность и вовлечённость).
- Фрейминг оплаты: «полный трекер + **поддержи своего учителя**» (rev-share делает оплату прямой пользой
  учителю — уникальный для этой модели драйвер).
- Расширение набора (аналитика, темы, инсайты дневника) — fast-follow после валидации.

### 6.4 Платёж (подписка)
- Бот выставляет **подписочный счёт Stars** (`subscription_period = 30 дней`, валюта XTR).
- Регулярный `successful_payment` → Edge Function `telegram-webhook` → upsert `entitlements`
  (`premium=true`, `premium_until = +период`, `plan='monthly'`, `provider='telegram'`); идемпотентно по
  charge id.
- Истечение/отмена: `premium_until` проходит → `usePremium()` отдаёт «не premium» (гаснет естественно).

### 6.5 Rev-share
- Каждый платящий ученик привязан к учителю (`referred_by_teacher`). Учитель получает **X%** (дефолт 40%)
  от **нетто** (после комиссии Telegram ~30%) с подписок своих учеников.
- Считается запросом: активные подписки за период × `referred_by_teacher`. **Выплата вручную** (MVP).

## 7. Данные и приватность
Новые/изменённые сущности:
- `teachers { id uuid PK = auth.uid, referral_code text unique, program jsonb, created_at }`
- `user_data += referred_by_teacher uuid null` (ставится один раз на входе по ссылке)
- `group_adherence { student_id uuid, teacher_id uuid, date date, done int, total int, streak int }` —
  **только числа, без содержимого дневника**
- `entitlements { id uuid PK, premium bool, premium_until timestamptz, plan text, provider text,
  updated_at }` (из Фазы 0)

**RLS / приватность:**
- `teachers`: владелец читает/пишет своё.
- `group_adherence`: ученик пишет/обновляет свои строки; учитель `SELECT` где `teacher_id = auth.uid`.
- `entitlements`: владелец `SELECT`; запись — **только** service-role Edge Function (клиент не выставит
  себе premium). Статус «платит/нет» своих учеников учитель видит через **безопасное представление**
  `teacher_dashboard` (отдаёт только `{student_id, paid bool, adherence}` для учеников этого учителя) —
  не полный entitlement.
- **`user_data.diary` НЕ раскрывается учителю никогда.** Гарантия: учитель видит дисциплину практик +
  факт оплаты, и ничего из личного дневника/контента.

## 8. Переиспользуем / строим
**Reuse (уже есть в приложении):** трекер, практики, `EditorSheet`, Supabase Auth, `user_data`.

Сборка естественно делится на **два этапа** (для плана реализации их стоит секвенировать):

**Этап 1 — платёжный фундамент (дизайн Фазы 0, ещё НЕ реализован):** Telegram Mini App + `telegram-auth`,
Stars-подписка, таблица `entitlements` + RLS, `telegram-webhook`, хук `usePremium()`, Paywall, гейт
«безлимит практик». Результат: ученик может подписаться, фича гаснет/зажигается. *(Можно запустить и без
учительского слоя — как обычный consumer-MVP.)*

**Этап 2 — слой «учитель-амбассадор»:** режим учителя + `referral_code`; атрибуция на входе; дашборд;
рекомендованная программа + инъекция в «Сегодня»; `group_adherence`; представление `teacher_dashboard`;
запрос rev-share; напоминания через бота.

## 9. План валидации
Найти 1–3 учителей цигун/медитации (личные контакты / профильные сообщества) → подключить их учеников →
3–4 недели → замерить: учителя реально шарят ссылку и открывают дашборд; ученики заходят и подписываются;
rev-share воспринимается как стоящий (интервью). **Гейт решения** (см. §2).

## 10. Риски и снижение
- **Учителя не продвигают** (боязнь «коммерциализации» отношений) → сначала интервью; дашборд должен быть
  реально полезным; фрейминг «помоги ученикам практиковать».
- **Ученики не платят** → убедительный гейт + «поддержи учителя» + низкая цена.
- **Утечка приватности (дневник)** → строгий RLS + представление; дневник не раскрываем — явно в дизайне.
- **Накрутка атрибуции** → first-touch, один учитель на ученика, фиксируется на первом входе.
- **Только Telegram-охват** → приемлемо для MVP (ученики в РФ в основном в Telegram).
- **Сложность Stars-подписки** (продления/отмены) → опираемся на регулярные события Telegram; entitlement
  на основе `premium_until` гаснет сам.

## 11. Тестирование
- **Юнит:** валидация HMAC `initData`; upsert entitlement (вкл. продление подписки); гейтинг
  `usePremium()`; запрос rev-share; запись `group_adherence`; **RLS** (учитель не читает дневник и чужих
  учеников).
- **E2E:** реальная тестовая Stars-подписка на живом Mini App → проверка entitlement + дашборда +
  выполнения + атрибуции.

## 12. Открытые параметры (дефолты, можно поправить при планировании)
- Цена подписки в Stars (≈ ₽199–299/мес эквивалент) — подобрать.
- Доля rev-share (дефолт 40% от нетто).
- Лимит бесплатных кастомных практик (дефолт 10).
- Период подписки (30 дней).
